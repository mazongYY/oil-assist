# Oil Assist - 油耗助手

## Project Overview
- **Type**: React SPA for fuel consumption tracking
- **Deploy target**: Cloudflare Pages (frontend) + Cloudflare Workers (API)
- **Data storage**: Cloudflare D1 (SQLite) via Worker API
- **Build**: Vite + TypeScript + TailwindCSS v4

## Architecture
- `src/types.ts` — Vehicle, FuelRecord interfaces (snake_case)
- `src/api.ts` — API client (fetch + JWT token)
- `src/utils.ts` — Stats calculation, chart segment computation
- `src/components/Layout.tsx` — Shell: mobile bottom nav + desktop sidebar (lg:)
- `src/pages/` — Dashboard, Vehicles, VehicleForm, VehicleDetail, Records, RecordForm, Login, Register
- `worker/` — Hono API (JWT auth + D1 CRUD)

## Routes
- `/login` — Login page
- `/register` — Register page
- `/` — Dashboard (stats overview)
- `/vehicles` — Vehicle list
- `/vehicles/new` — Add vehicle
- `/vehicles/:id` — Vehicle detail + chart + records
- `/vehicles/:id/edit` — Edit vehicle
- `/records` — All fuel records
- `/records/new` — Add record (supports `?vehicleId=` param)
- `/records/:id/edit` — Edit record

## Key Decisions
- D1 + Worker for multi-device sync (replaced localStorage)
- JWT auth with username/email login
- snake_case for API fields (vehicle_id, is_full, total_cost)
- Mobile-first responsive design with desktop sidebar
- Recharts for trend visualization
- `_redirects` for SPA routing on Cloudflare Pages
